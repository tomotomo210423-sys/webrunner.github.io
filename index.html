Web File Runner 徹底活用ガイド

プログラミングの学習・開発環境へようこそ

本ツールは、HTML/CSS/JavaScriptによるWebアプリケーションの開発を、サーバーを構築することなくブラウザ上だけで完結させるための統合開発環境（IDE）です。

1. Webアプリケーションを構成する「3つの言語」

優れたWebアプリは、役割の異なる3つの言語が連携して動いています。ファイルを分割して開発することで、コードの可読性が格段に上がります。

📄 HTML (UIの構造と骨組み)

画面に「ボタン」や「キャンバス（描画エリア）」「テキスト」をどこに配置するかを定義するマークアップ言語です。プログラムの土台となります。

🎨 CSS (デザインとレイアウト)

HTMLで作られた要素に対し、色、サイズ、余白、アニメーションなどの「装飾」を適用します。スマートフォンの画面サイズに合わせるレスポンシブ対応もここで行います。

⚙️ JavaScript (動的なロジックと演算)

「ユーザーが画面をタップしたときの処理」や「キャラクターの物理演算（重力や摩擦）」「スコアの計算」など、アプリの裏側で動く高度な処理をプログラムします。

2. 本格的な開発をサポートする「IDE機能」

Ver.1.7では、プロのエンジニアが使うエディタ「VS Code」と同じエンジン（Monaco Editor）を搭載しています。

  • 自動補完（オートコンプリート）: コードを途中まで入力すると、正しい関数の候補が表示されます。Tabキーで入力を省略でき、タイピングミスを防ぎます。
  • リアルタイム構文チェック: 括弧の閉じ忘れや、存在しない変数を指定した際に、赤い波線でエラー箇所を教えてくれます。
  • セーフティ・コンソール: JavaScriptの console.log() で出力した変数のデータや、プログラムの裏側で起きたエラーを安全にキャッチして右下に表示します。ゲームのタッチ操作を妨害しないよう、プレビュー画面とは完全に分離されています。（不要な時は「▼」ボタンで折りたためます）

3. アセット（画像・音声）の組み込み方

ローカルにある画像（.png, .jpg）や音声（.mp3）をゲーム内で使うためのテクニックです。

  • 手順: ソースコードのファイルと一緒に、使いたい画像や音声ファイルも「ファイルを開く」からまとめて一緒に読み込みます。
  • コードでの指定: HTMLやJavaScriptの中では、ファイル名（例: player.png）をそのままパスとして記述してください。
  • 裏側の仕組み: ツールが実行時に、アセットファイルを「Data URL（Base64という文字列）」に自動変換し、コード内のファイル名とすり替えるため、サーバーがなくても画像が表示されます。
